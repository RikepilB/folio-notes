## What Was Built

### Phase 1 — Core Features ✅
- CRUD operations for notes (create, read, update, soft-delete)
- Archive / unarchive notes
- Soft-delete trash with 30-day expiry + restore
- Real-time case-insensitive search
- TypeScript strict mode end-to-end

### Phase 2 — Extra Credit ✅
- Add / remove categories on individual notes
- Filter notes list by category
- Sort notes by creation date or update date (asc/desc)

## Architecture Notes

### Security (Current Demo State)
- **Frontend:** Public (no auth required)
- **Backend:** Public demo API (no verify-jwt) - suitable for demo/prototyping ONLY
- **Database:** RLS enabled but allow-all policies (demo mode)

### For Production
To secure for production:
1. Enable Supabase Auth
2. Add user_id column to notes
3. Enable JWT verification on Edge Function
4. Add RLS policies per user
5. Add rate limiting

## API Endpoints

### Notes
| Method | Path | Description |
|--------|------|-------------|
| GET | /notes | List with filters |
| POST | /notes | Create note |
| GET | /notes/:id | Get single note |
| PUT | /notes/:id | Update note |
| DELETE | /notes/:id | Soft delete |
| PATCH | /notes/:id/archive | Toggle archive |
| PATCH | /notes/:id/restore | Restore from trash |

### Categories
| Method | Path | Description |
|--------|------|-------------|
| GET | /categories | List all |
| POST | /categories | Create |
| DELETE | /categories/:id | Delete |
